## 二、Phase 0：地基重塑（4 周）

### 2.1 目标

把现有代码升级为"具备工程化基础"的状态，为后续所有工作铺底。

### 2.2 工作项（WBS）

#### W0.1 项目结构重组（3 天）

- [ ] 拆分 crate：`agent-core` / `agent-protocols` / `agent-platform` / `agent-telemetry` / `agent-cli` 等
- [ ] 制定 workspace 依赖管理规范（统一版本管理）
- [ ] 整理目录结构与现有代码迁移到新布局
- [ ] **交付物**：新的 workspace 目录树、迁移说明文档

#### W0.2 错误处理统一（3 天）

- [ ] 选择错误处理方案（`thiserror` + `anyhow` 组合）
- [ ] 定义统一错误类型：`AgentError`（库错误）、`AgentResult<T>`
- [ ] 错误码分级与命名规范（领域前缀 + 数字编码）
- [ ] 重构现有代码使用统一错误体系

## 三、Phase 1：安全加固（6 周）

### 3.1 目标

构建完整的安全基础：传输加密、身份认证、命令安全、审计追溯。

### 3.2 工作项

#### W1.1 Security Center 核心（5 天）

- [ ] 设计 `Security Center` 模块结构
- [ ] 证书加载与验证（`rustls` + `webpki`）
- [ ] 签名验证（Ed25519，`ring` 或 `ed25519-dalek`）
- [ ] 加解密接口（AES-GCM / ChaCha20-Poly1305）
- [ ] 密钥轮换接口
- [ ] **交付物**：`agent-core/src/security/` 模块

#### W1.2 KeyStore 平台抽象（5 天）

- [ ] `KeyStore` trait 定义
- [ ] Linux 实现：TPM 2.0（`tss-esapi`）+ 加密文件 fallback
- [ ] Windows 实现：DPAPI / TBS
- [ ] macOS 实现：Keychain
- [ ] 嵌入式 fallback：设备指纹绑定的加密文件
- [ ] 单元测试 + 集成测试
- [ ] **交付物**：KeyStore 各平台实现

#### W1.3 mTLS 全通道（7 天）

- [ ] 北向 gRPC Server 升级为 mTLS（`tonic` + `rustls`）
- [ ] MQTT 客户端升级为 mTLS（`rumqttc` + `rustls`）
- [ ] 证书加载策略（从 KeyStore）
- [ ] 证书过期监控与告警
- [ ] 自动证书轮换流程（先实现拉取，更新后续做）
- [ ] **交付物**：mTLS 通道、证书管理流程

#### W1.4 设备身份与注册（5 天）

- [ ] 设备唯一标识方案（hardware fingerprint + 证书）
- [ ] 首次注册流程（bootstrap with provisioning token）
- [ ] 证书申请（CSR 生成与提交）
- [ ] 设备身份持久化（KeyStore）
- [ ] **交付物**：设备注册流程文档与实现

#### W1.5 命令白名单与沙箱（7 天）

- [ ] 命令白名单设计（模板 + 参数 schema）
- [ ] 参数校验框架（正则 + 类型 + 范围）
- [ ] `SandboxRunner` trait 定义
- [ ] Linux 实现：seccomp + namespace + cgroup
- [ ] Windows 实现：JobObject + AppContainer（可降级）
- [ ] 命令执行超时与资源限制
- [ ] **交付物**：命令白名单配置 + Sandbox 实现

#### W1.6 RBAC 与责任链（5 天）

- [ ] 简易 RBAC 模型（角色 + 权限 + 资源）
- [ ] 责任链框架（拦截器/中间件模式）
- [ ] 标准拦截器：AuthN、AuthZ、RateLimit、Audit、Validate
- [ ] 应用到北向 gRPC 与南向 gRPC（南向待 Phase 2）
- [ ] **交付物**：RBAC 模块、拦截器链

#### W1.7 审计链（Audit Chain）（5 天）

- [ ] 审计日志表设计（SQLite）
- [ ] 哈希链实现（前一条 hash 作为下一条输入）
- [ ] 审计事件分类与字段规范
- [ ] 审计日志上报到后端（批量、压缩）
- [ ] 审计日志查询接口（本地与远程）
- [ ] **交付物**：Audit Chain 模块

#### W1.8 文件传输安全（4 天）

- [ ] 路径白名单与穿越防护
- [ ] 文件大小限制、磁盘配额检查
- [ ] 上传文件签名验证（可选）
- [ ] 分块 SHA-256 + 整体 SHA-256 双重校验
- [ ] 断点续传与持久化任务状态
- [ ] **交付物**：升级版 `File Transfer Service`

#### W1.9 安全测试（4 天）

- [ ] 渗透测试用例（命令注入、路径穿越、TLS 降级）
- [ ] Fuzz 测试关键解析逻辑（manifest 解析、命令参数）
- [ ] 模拟攻击场景的集成测试
- [ ] **交付物**：安全测试套件、测试报告

### 3.3 关键里程碑

| 时间  | 里程碑                          |
| ----- | ------------------------------- |
| W2 末 | Security Center + KeyStore 完成 |
| W3 末 | mTLS 全通道打通                 |
